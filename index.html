HTMLess vs Sanity

Sanity is faster today on delivery. HTMLess is stronger on control.

That is the honest tradeoff. HTMLess is not trying to copy Sanity's whole stack. It is keeping PostgreSQL as the source of truth and adopting the delivery patterns that matter most next.

Where Sanity Leads

Delivery and realtime are more mature today

Sanity still has a real edge in CDN-native delivery, GROQ-based reference resolution, listener APIs, and the kind of realtime editing experience teams expect from a polished SaaS platform.

  • CDN-first delivery API
  • GROQ resolves references in one query
  • Realtime listeners instead of polling
Where HTMLess Wins

Open control, SQL integrity, and easier ownership

HTMLess is the better fit for teams that want to self-host, avoid per-seat pricing, keep content in PostgreSQL, and use a simpler API and operational model they can fully own.

  • MIT-licensed open core
  • Self-hostable with Docker
  • PostgreSQL-backed relational integrity
What HTMLess Ships Next

Phase 9 is aimed directly at the gap

HTMLess is not ignoring what Sanity does well. The next delivery phase is focused on read-model speed, batched reference resolution, cache invalidation, and realtime preview/admin updates.

  • `published_documents` + denormalized CDA reads
  • Redis caching and CDN invalidation flows
  • SSE or WebSocket updates for preview and admin
